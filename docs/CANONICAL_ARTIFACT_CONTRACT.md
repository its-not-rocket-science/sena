# Canonical Artifact Contract

This document defines the supported replay contract for canonical artifacts on the
Jira + ServiceNow production path.

## Contract summary

- **Determinism scope is canonical only (`canonical_replay_payload_only`).**
  - `canonical_replay_payload` is replay-stable and deterministic.
  - `operational_metadata` is intentionally volatile and runtime-specific.
- **No supported surface should imply full raw response determinism.**
  - Equality checks must use canonical fields.
  - Runtime-only fields (timestamps, delivery/request IDs, write-time chain positions) must remain outside canonical payloads.

## Canonical vs. operational boundaries

## Canonical determinism_contract shape

Every supported artifact that claims replay stability should expose:

```json
{
  "determinism_contract": {
    "scope": "canonical_replay_payload_only",
    "canonical_replay_payload": { "...": "..." },
    "operational_metadata": { "...": "..." },
    "canonical_replay_payload_hash": "<sha256 canonical_replay_payload>"
  }
}
```

Notes:
- `canonical_replay_payload_hash` is SHA-256 over canonical JSON (sorted keys, compact separators).
- Some artifacts also include `volatile_fields` for explicit exclusion documentation.
- Legacy `determinism_scope` fields may appear for compatibility, but `determinism_contract`
  is the canonical vocabulary.

## Canonical replay payload (stable)

Use for replay parity checks, drift detection, and externally verifiable artifact exchange.

Typical canonical values include:
- proposal/action envelope,
- policy bundle identity,
- deterministic outcome data (`outcome`, matched rule/invariant IDs),
- deterministic provenance (`input_fingerprint`, `decision_hash`).

## Operational metadata (volatile)

Use for observability and runtime operations only.

Typical operational values include:
- event ingestion timestamps,
- delivery/request IDs generated by transport/runtime,
- write-time chain linkage fields and storage sequence placement.

## Supported surfaces that follow this contract

- Evaluate API and CLI decision traces.
- Replay export artifacts.
- Normalized integration webhook envelopes.
- Audit evidence bundle exports.
- Decision review package exports.

## Guardrails

- Prefer explicit `determinism_contract` markers in exported payloads.
- Keep canonical hashes derived from canonical payload content only.
- Keep docs and tests aligned with this boundary.
