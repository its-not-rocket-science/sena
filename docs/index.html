What it does

Sena evaluates approval requests against policy bundles and produces decisions that are deterministic, replayable, and auditable. The same request evaluated against the same policy bundle always produces the same decision — and that decision carries a complete, tamper-evident audit trail.

Designed for enterprise workflows where governance teams need to demonstrate to auditors exactly why a request was approved or rejected, and reproduce that reasoning months later.

Policy anatomy

Each policy is a structured bundle evaluated deterministically:

Subject Who the policy applies to
Condition When it triggers
Action What is permitted or required
Constraint Limits on the action
Exception Override conditions
Valid until Temporal scope

Audit chain

Request

Approval request received from Jira or ServiceNow

Evaluate

Policy bundle evaluated deterministically

Decision

Approved / Rejected / Escalated with full reasoning

Seal

Decision chained, signed, and stored as replayable evidence

Integrations

Jira

Webhook-driven approval evaluation. Policy decisions written back as comments and status transitions.

ServiceNow

REST integration for ITSM approval workflows. Full change request lifecycle support.

Audit replay

Replay any historical decision with the policy bundle active at the time. Produces byte-identical output.

Quick start

pip install sena

from sena import PolicyEngine, PolicyBundle

engine = PolicyEngine()
bundle = PolicyBundle.from_yaml("policies/change-management.yaml")

decision = engine.evaluate(request=request, bundle=bundle)
print(decision.outcome)      # APPROVED / REJECTED / ESCALATED
print(decision.audit_trail)  # replayable evidence chain
